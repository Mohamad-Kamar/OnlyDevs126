# Seed bibliography

These are high-signal anchors for synthesis, not a substitute for checking the linked text before quoting it.

## Systems thinking and dynamics

| Source | Useful for |
| --- | --- |
| [Donella Meadows, *Leverage Points: Places to Intervene in a System*](https://donellameadows.org/archives/leverage-points-places-to-intervene-in-a-system/) | A hierarchy of interventions; why changing a number is often lower leverage than changing information, rules, goals, or paradigms. |
| [Donella Meadows, *Thinking in Systems*](https://www.chelseagreen.com/product/thinking-in-systems/) | Accessible vocabulary for stocks, flows, feedback, delays, and boundaries. |
| [System Dynamics Society: What is System Dynamics?](https://systemdynamics.org/what-is-system-dynamics/) | A concise field definition and modeling orientation. |
| [Jay W. Forrester, *Industrial Dynamics*](https://mitpress.mit.edu/9780262560188/industrial-dynamics/) | Classic origin of feedback-based dynamic modeling. |
| [John Sterman, *Business Dynamics*](https://mitpress.mit.edu/9780262694647/business-dynamics/) | Rigorous treatment of dynamics, policy resistance, and model discipline. |

## MIT system dynamics and software project work

| Source | Useful for |
| --- | --- |
| [MIT OCW, 15.871 *Introduction to System Dynamics*](https://ocw.mit.edu/courses/15-871-introduction-to-system-dynamics-fall-2013/) | Foundational method: feedback, stocks and flows, delays, simulation, and testing policy interventions. The published Fall 2013 schedule is not evidence of a software-specific module. |
| [MIT OCW, ESD.36 *System Project Management*: Case 3, Software Project](https://ocw.mit.edu/courses/esd-36-system-project-management-fall-2012/pages/lecture-notes/) | Direct evidence that MIT taught a software-project case alongside project dynamics. Use as course provenance, not as empirical proof about all software projects. |
| [MIT OCW, ESD.36 Lecture 07: *The Rework Cycle*](https://ocw.mit.edu/courses/esd-36-system-project-management-fall-2012/resources/mitesd_36f12_lec07/) | Project rework, undiscovered rework, and discovery delay; supports asking when work was reported done versus confirmed end-to-end. |
| [MIT OCW, ESD.36 Lecture 08: *Project Dynamics: Feedbacks*](https://ocw.mit.edu/courses/esd-36-system-project-management-fall-2012/resources/mitesd_36f12_lec08/) | How staffing, experience, communication, fatigue, quality, and rework can interact in a simulation model. Do not attribute its cited external survey to MIT. |
| [John D. Sterman, *System Dynamics Modeling for Project Management*](https://web.mit.edu/jsterman/www/SDG/project.html) | MIT Sloan framing of software-project feedback, late discovery, rework, overtime, and staffing as dynamic, testable mechanisms; a 1992 source, not current prevalence data. |

## Citation discipline

Use these sources to support concrete claims; do not use them to launder a metaphor into a fact. Attribute quotations exactly, distinguish theory from a proposed engineering analogy, and include a caveat where a model is only a useful lens.
